Deployment

Deploy Quackbase to Cloudflare with D1, media storage, and optional services.

Core runtime

Cloudflare Workers execute the application and API logic close to users.

Structured data

Cloudflare D1 stores content, settings, sessions, forms, and administrative records.

Media delivery

R2 or another S3-compatible provider stores uploaded assets referenced by the CMS.

Optional integrations

Resend powers transactional email and Cloudflare Turnstile protects forms and protected actions.

Deployment checklist

  1. 1

    Provision Worker and D1

    Create the Worker runtime and attach the D1 database binding used by the application.

  2. 2

    Apply environment configuration

    Add runtime variables and secrets required by authentication, backup, media, and email workflows.

  3. 3

    Configure optional services

    Only enable R2, Turnstile, or Resend when your deployment needs those capabilities.

  4. 4

    Validate the admin dashboard

    Confirm that login, content reads, and settings screens load correctly in the deployed environment.

Secret handling

Keep API tokens, storage keys, and email credentials in environment secrets or encrypted admin storage. Do not hardcode them into static assets or client-side code.
