Getting Started

Deploy

Quackbase is happiest on Cloudflare. The fastest path is the deploy button, then a few admin-side configuration steps.

Cloudflare one-click path

  1. Open the deploy button from the repository.
  2. Create or connect the required Cloudflare resources.
  3. Confirm the Worker deployment.
  4. Open the deployed domain and visit /admin.
  5. Finish the initial admin setup.
Deploy link
https://deploy.workers.cloudflare.com/?url=https://github.com/xuanlockun/astro-blog-starter-template

What should be ready after first deploy

  • D1 binding and bootstrap migration applied.
  • Admin account created from the setup flow.
  • Site settings reviewed inside the dashboard.
  • Optional integrations like R2, Resend, and Turnstile configured as needed.

Optional local validation

Checks
npm run check
npm run deploy

Before going public

  • Upload a favicon and verify site branding.
  • Set Resend email if forms should send notifications.
  • Enable captcha if the public form is exposed.
  • Export a backup once the initial content and settings are in place.
