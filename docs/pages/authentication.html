Architecture→Authentication
Security

Authentication

Protect admin routes with sessions and role-based access checks.

Session-based admin access

Administrative routes should require an authenticated session before rendering sensitive data or mutating state.

Role-based authorization

Roles and permissions should control access to settings, user management, backup operations, and administrative APIs.

Credential boundaries

Secrets, API tokens, and service keys must stay on the server and never be exposed in public responses or frontend code.

Operational hardening

Protected actions such as backup, restore, or settings mutation should be gated by both authentication and explicit permission checks.
