Languages

Quackbase already has the pieces for multilingual UI and localized content, including supported language management and translation entry workflows.

What is localized

  • UI labels and navigation copy
  • Posts and pages
  • Localized slugs
  • Some shared field labels such as contact form labels

How it works

  1. Create or enable languages in the admin.
  2. Choose one default language.
  3. Fill translated values for content and translation entries.
  4. Let the site fall back to the default language when a translation is missing.

Why this matters

Better editorial control

Editors manage language variants in one place instead of duplicating sites or content sources.

Cleaner public URLs

Localized slugs and language-aware routing make the frontend feel natural for multilingual visitors.
