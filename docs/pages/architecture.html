Architecture

Edge Stack

Quackbase is built for a Cloudflare edge stack: Astro frontend, Cloudflare Workers runtime, D1 for content data, and R2 or another S3-compatible store for media.

Main pieces

Cloudflare Workers

Runs the Astro app and admin APIs at the edge.

Cloudflare D1

Stores posts, pages, languages, settings, sessions, forms, roles, and permissions.

Cloudflare R2

Recommended for uploads and media files, using the media settings in the admin.

Turnstile and Resend

Turnstile protects forms from spam and Resend handles email delivery.

How it fits together

Stack overview
Astro frontend
-> Cloudflare Workers runtime
-> D1 for content and settings
-> R2 or S3-compatible media storage
-> Turnstile for captcha
-> Resend for email delivery

Why this setup

No traditional server is required. The project stays close to Cloudflare-native infrastructure and avoids extra platform layers that do not help publishing.
