Configuration

Captcha

Quackbase includes captcha support so public forms can reject obvious spam before submissions hit your content workflow.

Typical setup

  1. Create a Cloudflare Turnstile site key and secret.
  2. Store the values through the admin or your runtime secret flow.
  3. Enable captcha on the contact form or public form surface that needs protection.
  4. Submit a test form to verify challenge and server validation both pass.

Why enable it

Spam reduction

Stops noisy bot traffic before it fills form submissions and inbox notifications.

Cleaner operations

Keeps the admin leads view and email notifications focused on real messages.

Validation flow

High-level flow
Visitor opens contact form
-> completes captcha challenge
-> form submits to /api/forms/contact
-> server verifies captcha token
-> valid submissions continue to save / notify
