Backup & Restore

Plan data recovery for Cloudflare-based deployments.

Status

Backup and restore workflows require Cloudflare API access. Full setup instructions will be documented later.

What can be backed up

D1 database exports

Relational content and application records can be exported from Cloudflare D1.

R2 media and metadata

Media objects or object metadata can be included depending on the storage workflow you implement.

Application configuration

Configuration state can be captured separately from the live runtime when you design the backup workflow.

Environment settings

Environment configuration should be tracked carefully, excluding secrets unless they are handled through a secure encrypted process.

Required Cloudflare access

Account ID

Needed to target the correct Cloudflare account for automation.

API token

Use a scoped token with the minimum required permissions for backup tasks.

Resource permissions

Grant access only to the D1 databases, R2 buckets, and other resources involved in backup or restore.

Target resources

Define the exact D1 database and R2 bucket boundaries used by the deployment.

Recommended backup flow

flowchart LR
  A["Schedule / Manual Trigger"] --> B["Cloudflare API"]
  B --> C["Export D1"]
  B --> D["Export R2 Metadata / Objects"]
  C --> E["Store Backup Artifact"]
  D --> E
  E --> F["Verify Backup"]

Restore flow

flowchart LR
  A["Select Backup"] --> B["Validate Artifact"]
  B --> C["Restore D1"]
  B --> D["Restore R2 Objects"]
  C --> E["Verify App Health"]
  D --> E

Security note

Avoid storing API tokens or secrets in plain text. Use encrypted settings, secret stores, or environment secrets instead.
