Configuration

Email

SMTP support in Quackbase is useful for contact form notifications and other operational messages that should leave the CMS automatically.

What to configure

  • SMTP host
  • Port
  • Username
  • Password or app secret
  • From address
  • Destination inbox for notifications

Recommended flow

  1. Enter SMTP credentials in the admin settings.
  2. Run the built-in SMTP test endpoint or admin test action.
  3. Submit a real form once to confirm notification delivery.
  4. Check spam handling and sender reputation for production domains.
Why it matters
Public form submit
-> validate captcha and fields
-> store submission
-> send notification email
-> admin team receives lead
