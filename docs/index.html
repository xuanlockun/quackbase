Why this exists

Quackbase is for teams who want something small, hackable, and easy to reason about. Instead of bolting a generic CMS onto Astro, it keeps the stack close to the site: Astro on the frontend, Workers at the edge, D1 for content, R2 for media, and admin tools focused on practical publishing.

What feels different

The project is built around direct deployment, runtime updates, and focused admin capabilities like media, language management, backup through Cloudflare D1 APIs, captcha, Resend email, and content operations.

Best fit

Blogs, landing pages, docs sites, startup websites, changelogs, and small products that want CMS power without infrastructure overhead.

What you get

The product stays focused on the workflow most small teams actually need: publish content, manage assets, configure the site, and keep the whole thing easy to operate.

Clean admin UI

Manage posts, pages, users, roles, languages, banners, contact forms, theme sections, and site settings from one dashboard.

Cloudflare-native deployment

Built for Cloudflare Workers, D1, and R2, with a deployment path that stays simple from first launch to production.

Operational tooling

Backup export/import, media sync, Resend email test flows, secret verification, and runtime configuration helpers are already in the repo.

Easy to extend

The codebase stays understandable with Astro pages, TypeScript libraries, and focused modules for auth, media, forms, i18n, and edge integrations.
